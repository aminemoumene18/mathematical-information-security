1. Цель работы

Изучить метод Полларда (ро-метод) для решения задачи дискретного логарифмирования в конечном поле и реализовать соответствующий алгоритм программно.

2. Теоретическая часть

Задача дискретного логарифмирования формулируется следующим образом: для заданных простого числа ( p ), образующего элемента ( a ) и элемента ( b ) найти целое число ( x ) такое, что:

[
a^x \equiv b \pmod{p}
]

Число ( x ) называется дискретным логарифмом и обозначается ( x = \log_a b ).

Для решения задачи был применен ρ-метод Полларда, использующий случайное отображение ( f ), которое позволяет выражать логарифм текущего элемента через неизвестный ( x ). В реализации использовалось ветвящееся правило:

  • Если текущее состояние ( x \mod 3 = 0 ), то ( x \leftarrow x \cdot b \mod p ), ( \beta \leftarrow \beta + 1 ).
  • Если ( x \mod 3 = 1 ), то ( x \leftarrow x^2 \mod p ), ( \alpha \leftarrow 2\alpha ), ( \beta \leftarrow 2\beta ).
  • Если ( x \mod 3 = 2 ), то ( x \leftarrow x \cdot a \mod p ), ( \alpha \leftarrow \alpha + 1 ).

Метод основан на поиске коллизий с использованием «черепахи и зайца» (алгоритм Флойда).

3. Реализация

Программа написана на языке Python. В ходе работы были реализованы следующие функции:

  • pollard_rho_dlog(p, a, b, r, max_iter) — реализация ρ-метода Полларда.
  • verify_dlog(p, a, b, x) — проверка найденного значения.
  • Вспомогательная итерационная функция f.

Программа принимает на вход параметры ( p, a, b, r ) и возвращает значение ( x ), если оно найдено, либо сообщение об ошибке.
